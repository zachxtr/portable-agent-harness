## Session: Policy Profile — Documents, Rich Legislation, and UI Polish

**Date:** 2026-05-20
**Commits:** `5dd989eb` — policy profile updates, tags and handle, other todos completed

### Completed
- Users can now attach specific bill PDF documents (filed versions, amendments, analyses, committee votes) directly to a Policy Profile — the document appears alongside associated bills and statutes on the profile page, with a click-through that opens the PDF viewer
- The "Add to Policy Profile" button is now available inside the PDF viewer itself — so a user reading an analysis can add it to a profile without navigating back to the bill
- Policy profile pages no longer need to make extra network calls to display associated bills and statutes — the bill name, session, and statute section are stored at the moment the user adds them, making the profile detail and edit pages load faster and work offline-first
- The PDF viewer now shows the document's published date in both the page header and the sidebar — previously the date was only visible back on the bill detail page in the document list
- Completed the "handle" rename across the Policy Profile feature — profiles now display their short identifier with an `@` prefix (e.g. `@FloridaEducation`), consistent with the social media convention established for the platform
- All tag displays throughout the library, saved items, and team activity pages now show a `#` prefix (e.g. `#Priority`, `#Watch`) — display only, consistent with how tags work on social platforms; no change to stored values
- If a user's profile settings file is missing from storage (e.g. after a restore or account reset), a fresh default is now created automatically on their next login — prevents errors when managing saved tags and preferences

### Next Steps
- Existing policy profiles in storage have associated bills and statutes stored as simple keys — a future pass could enrich those records with the full title and session data now captured for new associations, so older profiles display with the same detail as newly-created ones
- Policy Profile contacts feature — add allies and opponents as named contacts with social links
- Walk through the "Ask Assistant from Profile" flow so that a profile's associated bills and statutes feed the AI conversation as context
